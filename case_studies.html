// the_challenge

Maggie's needed comprehensive QA across their centre-finder tool, appointment booking system and online support resources — serving cancer patients and their families across the entire UK network. The audience is particularly vulnerable, making reliability and accessibility critical. The platform needed to work flawlessly across all devices and browsers their users would realistically access it from.

// my_approach

I developed a detailed set of test cases from the functional specifications, prioritising the postcode-based centre finder and appointment booking journeys as the most critical paths. Cross-browser and device testing was conducted using BrowserStack across 15 real device/OS combinations. Playwright was used to automate regression tests ensuring ongoing stability. Accessibility was tested against WCAG 2.1 AA throughout.

// engagement_timeline
Phase 1
Test case authoring from specifications
Produced 31 detailed test cases covering all functional areas, with clear pass/fail criteria and priority ratings.
Phase 2
Cross-browser & device testing via BrowserStack
Tested across Chrome, Firefox, Safari, Edge, iOS Safari and Chrome Android on 15 real device configurations.
Phase 3
Playwright automation for regression
Key centre-finder and booking journeys automated to protect against future regressions through ongoing releases.

// the_challenge

Bipolar UK's self-assessment form is used by people in genuinely vulnerable mental health situations. Any bugs, confusing UX or accessibility failures carry real human consequences. The brief required comprehensive QA with a sensitivity to the audience — ensuring every user journey was not only functional but safe, clear and accessible to people who may be in crisis or using assistive technologies.

// my_approach

I approached this engagement with particular care, conducting a thorough WCAG 2.1 AA audit alongside functional testing. Screen reader testing was performed using NVDA and VoiceOver. UAT sessions were structured to simulate real user behaviour including non-linear navigation, back-button usage and form abandonment patterns. All findings were documented with clear reproduction steps and impact assessments tailored to the charity's non-technical stakeholders.

// the_challenge

Breast Cancer UK needed thorough QA across their website and a high-stakes risk assessment form used by thousands of women to assess their personal cancer risk factors. The form integrates with a backend API and produces personalised results — meaning data integrity and API reliability were as important as the UI. Cross-browser consistency across 18 device and browser combinations was also a key requirement.

// my_approach

I structured the engagement in two parallel tracks — UI/functional testing and API testing. The assessment form was tested exhaustively including boundary value analysis on numeric inputs, validation edge cases, and API response handling. Postman was used to validate all API endpoints independently. Cross-browser and responsive testing was conducted using BrowserStack. Performance was assessed using Lighthouse and Chrome DevTools.

// engagement_timeline
Track A — UI Testing
Form validation & responsive testing
52 test cases covering all form paths, validation states, responsive breakpoints and cross-browser behaviour.
Track B — API Testing
Endpoint validation & response schema testing
All form submission endpoints validated via Postman — checking response schemas, error handling and data integrity.
Track C — Performance
Lighthouse & Core Web Vitals audit
Performance audit identifying render-blocking resources and CLS issues, resolved before launch.
